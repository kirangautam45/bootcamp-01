Notes App - Frontend Only Setup Guide

NO BACKEND REQUIRED

This guide will walk you through setting up a frontend-only React Notes application. This version stores notes in browser memory (React state) - no database or server needed!

Why Frontend-Only?

This is perfect for:

  • Learning React fundamentals without backend complexity
  • Quick prototyping and testing UI ideas
  • Understanding state management before adding a database

Technology Stack

React 19 JavaScript library for building UIs
Vite Fast build tool & dev server
Lucide React Beautiful icon library

Code Explanation

1. main.jsx - Entry Point

This is where React mounts your app to the DOM:

import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>
)

Line-by-Line:

  • StrictMode: Helps find potential problems during development (double-renders components)
  • createRoot: React 18+ API for creating a root to render into
  • document.getElementById('root'): Finds the div in index.html

2. App.jsx - Main Component (State Manager)

This is where all the data and logic lives:

import { useState } from 'react'
import NoteForm from './components/NoteForm'
import NoteList from './components/NoteList'
import './App.css'

function App() {
  // STATE - stored in memory (resets on page refresh)
  const [notes, setNotes] = useState([])           // Array of all notes
  const [editingNote, setEditingNote] = useState(null) // Note being edited
  ...
}
Key Difference: No Backend!

Notice there's no fetch() or API calls. All data is stored in React state (useState). When you refresh the page, notes disappear because they only exist in memory.

addNote Function - Create New Note

addNote(note)

Creates a new note with auto-generated ID and timestamp

const addNote = (note) => {
  const newNote = {
    id: Date.now(),              // Unique ID using timestamp
    ...note,                       // Spread title, content, color
    createdAt: new Date().toISOString()  // Current date/time
  }
  setNotes([newNote, ...notes])  // Add to FRONT of array
}

How It Works:

  • Date.now() returns milliseconds since 1970 - perfect for unique IDs
  • ...note spreads the incoming object (title, content, color)
  • [newNote, ...notes] puts new note first, then all existing notes

updateNote Function - Edit Existing Note

updateNote(id, updatedNote)

Finds and updates a note by its ID

const updateNote = (id, updatedNote) => {
  setNotes(notes.map(note =>
    note.id === id                     // Find the note with matching ID
      ? { ...note, ...updatedNote }    // Merge old + new properties
      : note                           // Keep other notes unchanged
  ))
  setEditingNote(null)  // Exit edit mode
}

How It Works:

  • map() creates a new array by transforming each element
  • When IDs match, spread both old and new properties (new overwrites old)
  • When IDs don't match, return the note unchanged

deleteNote Function - Remove Note

deleteNote(id)

Removes a note from the array

const deleteNote = (id) => {
  setNotes(notes.filter(note => note.id !== id))
}

How It Works:

  • filter() creates new array with only items that pass the test
  • note.id !== id keeps all notes EXCEPT the one being deleted

JSX Return (UI Structure)

return (
  <div className="app">
    <header className="header">
      <h1>Notes App</h1>
    </header>

    <main className="main">
      {/* Form for creating/editing notes */}
      <NoteForm
        key={editingNote?.id || 'new'}  // Forces remount on mode change
        onSubmit={editingNote
          ? (note) => updateNote(editingNote.id, note)
          : addNote}
        editingNote={editingNote}
        onCancel={() => setEditingNote(null)}
      />

      {/* List of all notes */}
      <NoteList
        notes={notes}
        onEdit={setEditingNote}  // Pass the setter directly
        onDelete={deleteNote}
      />
    </main>

    <footer className="footer">
      <p>Notes App - No Backend Required</p>
    </footer>
  </div>
)

The key Prop Trick:

key={editingNote?.id || 'new'}

This forces React to remount the NoteForm component when switching between add/edit modes. This resets the form's internal state automatically!

3. NoteForm.jsx - Create/Edit Form

import { useState } from 'react'
import './NoteForm.css'

// Color options for notes
const COLORS = [
  '#ffffff',   // White
  '#fff9c4',   // Yellow
  '#c8e6c9',   // Green
  '#bbdefb',   // Blue
  '#d1c4e9',   // Purple
  '#ffccbc'    // Orange
]

function NoteForm({ onSubmit, editingNote, onCancel }) {
  // Local state for form fields
  // Initialize with editing values OR empty defaults
  const [title, setTitle] = useState(editingNote?.title || '')
  const [content, setContent] = useState(editingNote?.content || '')
  const [color, setColor] = useState(editingNote?.color || '#ffffff')

  const handleSubmit = (e) => {
    e.preventDefault()  // Prevent page reload

    // Validation: require both title and content
    if (!title.trim() || !content.trim()) return

    // Call parent's onSubmit with note data
    onSubmit({ title, content, color })

    // Only clear form if adding (not editing)
    if (!editingNote) {
      setTitle('')
      setContent('')
      setColor('#ffffff')
    }
  }
  ...
}

Key Concepts:

Concept Example Purpose
Optional Chaining editingNote?.title Safely access properties (returns undefined if null)
Nullish Coalescing ?.title || '' Provide default value if undefined/null
Controlled Inputs value={title} React controls the input value via state
preventDefault() e.preventDefault() Stops form from reloading the page

Color Picker Implementation:

<div className="color-picker">
  {COLORS.map((c) => (
    <button
      key={c}                        // Unique key for React
      type="button"                  // Prevents form submit!
      className={`color-btn ${color === c ? 'selected' : ''}`}
      style={{ backgroundColor: c }} // Dynamic inline style
      onClick={() => setColor(c)}   // Update color state
    />
  ))}
</div>
Important: type="button"

Without type="button", clicking a color would submit the form! Buttons inside forms default to type="submit".

4. NoteList.jsx - Display Notes Grid

import NoteCard from './NoteCard'
import './NoteList.css'

function NoteList({ notes, onEdit, onDelete }) {
  // Empty state - show message when no notes
  if (notes.length === 0) {
    return (
      <div className="empty-state">
        <p>No notes yet. Create your first note!</p>
      </div>
    )
  }

  // Map through notes array and render NoteCard for each
  return (
    <div className="note-list">
      {notes.map((note) => (
        <NoteCard
          key={note.id}      // Required unique key
          note={note}        // Pass note data
          onEdit={onEdit}    // Pass edit handler
          onDelete={onDelete} // Pass delete handler
        />
      ))}
    </div>
  )
}

export default NoteList

CSS Grid Layout:

.note-list {
  display: grid;
  grid-template-columns: repeat(4, 1fr);  /* 4 equal columns */
  gap: 20px;                               /* Space between cards */
}

5. NoteCard.jsx - Individual Note Display

import { Pencil, Trash2 } from 'lucide-react'  // Icon imports
import './NoteCard.css'

function NoteCard({ note, onEdit, onDelete }) {
  // Helper function to format dates nicely
  const formatDate = (dateString) => {
    return new Date(dateString).toLocaleDateString('en-US', {
      month: 'short',   // "Dec"
      day: 'numeric',   // "11"
      year: 'numeric'   // "2025"
    })
  }

  return (
    <section
      className="note-card"
      style={{ backgroundColor: note.color || '#ffffff' }}
    >
      <h3 className="note-title">{note.title}</h3>
      <p className="note-content">{note.content}</p>

      <div className="note-footer">
        <span className="note-date">
          {formatDate(note.createdAt)}
        </span>

        <div className="note-actions">
          <button onClick={() => onEdit(note)} className="action-btn">
            <Pencil size={18} />
          </button>
          <button onClick={() => onDelete(note.id)} className="action-btn">
            <Trash2 size={18} />
          </button>
        </div>
      </div>
    </section>
  )
}

export default NoteCard

Lucide Icons Usage:

  • Import specific icons: { Pencil, Trash2 }
  • Use as components: <Pencil size={18} />
  • Customize with props: size, color, strokeWidth

Data Flow Diagram

1. Adding a Note:

User types → Form state updates → Submit → App.addNote() → setNotes() → UI re-renders

2. Editing a Note:

Click Pencil → onEdit(note) → setEditingNote() → Form fills with data → Submit → updateNote() → UI updates

3. Deleting a Note:

Click Trash → onDelete(id) → deleteNote() → filter() removes note → UI updates

Frontend-Only vs Full-Stack Comparison

This App (Frontend-Only)

  • Data stored in React state
  • Notes disappear on refresh
  • No server needed
  • Instant - no network latency
  • Great for learning React

Full-Stack Version

  • Data stored in MongoDB
  • Notes persist forever
  • Requires backend server
  • API calls add latency
  • Real-world architecture

Next Steps to Add Persistence

To make notes persist (survive page refresh), you can:

Option 1: localStorage (Easiest)

// Save to localStorage whenever notes change
useEffect(() => {
  localStorage.setItem('notes', JSON.stringify(notes))
}, [notes])

// Load from localStorage on mount
const [notes, setNotes] = useState(() => {
  const saved = localStorage.getItem('notes')
  return saved ? JSON.parse(saved) : []
})

Option 2: Add a Backend

Connect to a Node.js/Express backend with MongoDB for real persistence and multi-user support.

Troubleshooting

Notes Disappear on Refresh

This is expected! Data is only stored in memory. Add localStorage (above) or a backend to persist data.

Form Not Clearing After Edit

The key prop on NoteForm handles this. Make sure it changes when switching modes.

Port 5173 Already in Use

Vite will auto-select another port, or kill the process using that port.
